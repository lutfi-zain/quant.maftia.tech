## Requirements
### Requirement: 1:1 Parity of Summary Performance Cards against Canonical Python Backtest

The frontend `IchimokuTerminal.tsx` (`studioBacktest.ts`) SHALL compute and display summary performance metrics (Win Rate %, Profit Factor, Total Trades, Sharpe Ratio, Annualized Return %, Annualized Volatility %, Max Drawdown %, Total Return %) that match $1:1$ with `run_backtest(df)` from `quant-lttd-ichimoku/src/ichimoku_quant/backtest.py` across both Strategy and Market Hold modes for any given date slice (`2018-01-01` to `NOW()`).

To achieve this parity, the hook SHALL prioritize the authoritative `ichimoku_strat_net_ret` and `ichimoku_active_pos` fields from `DailyAnalyticsPoint` when computing metrics, falling back to position-based recomputation only when these fields are unavailable (pre-migration data).

#### Scenario: Annualized Strategy Return and Volatility match backtest.py

- **WHEN** the studio aggregates daily strategy returns over the selected window
- **THEN** the computed `annReturnStrat` and `annVolatilityStrat` SHALL match the exact scaling formula (`np.sqrt(365.25)` annualized factor and causal active $t-1$ position friction) from `backtest.py` with numeric precision difference $|a - b| < 10^{-6}$
- **AND** when `ichimoku_strat_net_ret` is available for all rows in the window, this SHALL be achieved by compounding the stored daily returns directly (identical to Python's `Strat_Net_Ret`)

#### Scenario: Win Rate, Profit Factor, and Total Trades exactly equal Python execution

- **WHEN** the studio backtest counts winning and losing trades
- **THEN** `winRate`, `profitFactor`, and `totalTrades` SHALL match the exact integer counts and ratios generated by `backtest.py` for the same window
- **AND** when `ichimoku_active_pos` is available, trade detection SHALL use position transitions from this field (identical to Python's `Active_Pos` diff)

### Requirement: 1:1 Parity of S-Components, Momentum Oscillator, and Candlesticks across Subplots
All 4 chart subplots (`btcChart`, `imoChart`, `scompChart`, `eqChart`) inside `IchimokuTerminal.tsx` SHALL display indicator values sourced directly from `unified_daily_analytics` via `/api/v1/quant/daily`, matching the exact numerical output of `quant-lttd-ichimoku` bar-by-bar without synthetic curve fitting or rounding drift.

#### Scenario: Subplot series values equal backend database values
- **WHEN** `IchimokuTerminal.tsx` renders price bars and indicator curves for a selected date window
- **THEN** every rendered point on `sTkSeries`, `sCloudSeries`, `sFutureSeries`, `sChikouSeries`, `imoSeries`, `entropySeries`, `erSeries`, and `imoStdSeries` SHALL match the corresponding `ichi_s_tk`, `ichi_s_cloud`, `ichi_s_future`, `ichi_s_chikou`, `ichimoku_imo`, `mttd_entropy`, `mttd_er`, and `0.40 * IMO_Std` values stored in `unified_daily_analytics`

### Requirement: 1:1 Parity of Trade Execution Table Rows

The frontend execution log (`trades`) generated in `studioBacktest.ts` SHALL output the exact identical sequence of trade entries and exits as `backtest.py`, including entry date, exit date, entry price, exit price, and return percentage.

#### Scenario: Trade log entries exactly mirror backtest transition states

- **WHEN** a position transition (`0 -> 1` or `1 -> 0`) occurs in `ichimoku_active_pos`
- **THEN** `studioBacktest.ts` SHALL push an entry to `trades` whose `entryPrice`, `exitPrice`, and `returnPct` equal `backtest.py`'s log within $|a - b| < 10^{-6}$
